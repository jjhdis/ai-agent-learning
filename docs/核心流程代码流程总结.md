## 1. 整体架构概览

```
┌─────────────┐      ┌─────────────────────────────────────────────┐
│   用户输入   │ ──→  │                 Agent.run()                 │
│ "上海天气"   │      │                                             │
└─────────────┘      │  ① 记录用户消息到 history                    │
                     │  ② 获取工具定义 → 传给 LLM                   │
                     │  ③ 进入 ReAct 循环                           │
                     │     ├─ LLM 推理                              │
                     │     ├─ 有 tool_calls? → 执行工具 → 继续循环   │
                     │     └─ 无 tool_calls? → 返回最终回复          │
                     └─────────────────────────────────────────────┘
                                │
                                ▼
                     ┌─────────────────────┐
                     │     LLM 客户端       │
                     │  (OpenAI 兼容接口)   │
                     └─────────────────────┘
                                │
                     ┌─────────────────────┐
                     │    Tool 注册中心      │
                     │  (管理所有工具)       │
                     └─────────────────────┘

                     ┌─────────────────────┐
                     │  具体工具 (Weather)   │
                     │  发 HTTP 请求查天气   │
                     └─────────────────────┘
```

### 第 0 步：启动准备（在 main.py 中）

### 第 1 步：`run()` 启动 —— 记录用户消息 & 获取工具定义

**代码位置**：`agent/core/agent.py` 第 47-60 行

```python
def run(self, user_message: str) -> str:
    self._emit("on_start", user_message)                          # 触发钩子

    self.history.append({"role": "user", "content": user_message}) # ① 记录用户消息
    tools = self.registry.get_openai_definitions()                 # ② 获取工具定义
```

#### ① 记录用户消息

```python
self.history.append({"role": "user", "content": "今天上海天气怎么样"})
```

此时 `self.history` 的内容：

```python
[
    {"role": "user", "content": "今天上海天气怎么样"}
]
```

> **📌 知识点：history 是什么？**
>
> `self.history` 是一个 `list[dict]`，按 **OpenAI Chat 消息格式** 存储对话历史。
> 每条消息都有 `role` 字段，有三种角色：
> - `"user"`：用户说的话
> - `"assistant"`：AI 说的话（可能包含 tool_calls）
> - `"tool"`：工具执行返回的结果
>
> **为什么要有 history？** 因为 LLM 本身是无状态的，每次调用都要把完整的对话历史传给它，它才能"记住"之前说过什么。

#### ② 获取工具定义

调用链：`registry.get_openai_definitions()` → 遍历所有已注册工具 → 调用每个工具的 `to_openai_function()`

```python
# registry.py
def get_openai_definitions(self) -> list[dict]:
    return [t.to_openai_function() for t in self._tools.values()]
```

`to_openai_function()` 把工具转换成 OpenAI 能看懂的 JSON 格式：

```python
# base.py
def to_openai_function(self) -> dict:
    properties = {}
    required = []
    for p in self.parameters:
        prop = {"type": p.type, "description": p.description}
        properties[p.name] = prop
        if p.required:
            required.append(p.name)

    return {
        "type": "function",
        "function": {
            "name": self.name,              # "get_weather"
            "description": self.description, # "查询指定城市的实时天气..."
            "parameters": {
                "type": "object",
                "properties": properties,    # {"city": {"type": "string", "description": "..."}}
                "required": required,        # ["city"]
            },
        },
    }
```

最终生成的 JSON：

```json
[
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "查询指定城市的实时天气和未来天气预报...",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "城市名称，中文或英文均可..."
                    }
                },
                "required": ["city"]
            }
        }
    }
]
```

> **📌 知识点：AI 是怎么看懂工具的？**
>
> 1. 开发者通过 `BaseTool` 定义工具的 **name / description / parameters**（相当于写说明书）
> 2. `to_openai_function()` 把说明书翻译成 **OpenAI Function Calling 格式的 JSON**
> 3. 这个 JSON 通过 OpenAI API 的 `tools` 参数传给 LLM
> 4. LLM **在训练时就已经见过无数类似的工具定义格式**，它能读懂 description 来理解工具用途，读懂 parameters 来知道需要什么参数
> 5. 当 LLM 觉得需要调用工具时，它在回复中返回 `tool_calls` 字段

### 第 2 步：进入 ReAct 循环 —— 第 1 轮推理

**代码位置**：`agent/core/agent.py` 第 62-74 行

```python
for step in range(self.max_iterations):       # max_iterations = 5
    msg = self.llm.chat(self.history, tools if tools else None)
```

调用 `LLMClient.chat()`：

```python
# client.py
def chat(self, messages, tools=None):
    kwargs = dict(
        model=self.config.model,
        messages=messages,          # 传入完整历史
        temperature=...,
        max_tokens=...,
    )
    if tools:
        kwargs["tools"] = tools     # 传入工具定义！
    
    response = self._client.chat.completions.create(**kwargs)
    return response.choices[0].message
```

**发送给 LLM 的内容**：

```
messages = [
    {"role": "user", "content": "今天上海天气怎么样"}
]

tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "查询指定城市的实时天气...",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {"type": "string", "description": "城市名称..."}
                },
                "required": ["city"]
            }
        }
    }
]
```

**LLM 的思考过程（模拟）**：

> 用户问"今天上海天气怎么样" → 我有一个工具叫 `get_weather`，它的 description 说可以查天气 → 它需要一个 `city` 参数 → 用户提到了"上海" → 我应该调用 `get_weather(city="上海")` 来获取天气数据

**LLM 返回的 `msg`**：

```
msg.content = None   # 没有文本回复，因为 LLM 决定调用工具
msg.tool_calls = [
    ChatCompletionMessageToolCall(
        id="call_abc123",
        function=Function(
            name="get_weather",
            arguments='{"city": "上海"}'
        )
    )
]
```

### 第 3 步：处理 LLM 返回 —— 判断是否要调工具

```python

```

`msg.model_dump()` 把 OpenAI 对象转成字典：

```python
{
    "role": "assistant",
    "content": None,
    "tool_calls": [
```


> **📌 知识点：为什么要把 tool_calls 也存进 history？**
>
> 因为 LLM 需要知道"自己之前已经调过哪些工具了"。如果不存，下一轮 LLM 会忘记自己调过工具，可能重复调用。

### 第 4 步：执行工具 —— `_handle_tool_calls()`

**代码位置**：`agent/core/agent.py` 第 114-136 行

```python
def _handle_tool_calls(self, tool_calls) -> None:
    results = []
    for tc in tool_calls:                              # 遍历每个工具调用
        tool = self.registry.get(tc.function.name)     # ① 找工具
        if not tool:
            content = f"[错误] 未知工具: {tc.function.name}"
        else:
            try:
                args = json.loads(tc.function.arguments) # ② 解析参数
            except json.JSONDecodeError:
                content = f"[错误] 参数解析失败: {tc.function.arguments}"
            else:
                self._emit("on_tool_call", tool.name, args)
                content = tool.execute(**args)           # ③ ★ 真正执行工具！

        results.append({                                 # ④ 收集结果
            "role": "tool",
            "tool_call_id": tc.id,
            "content": content,
        })

    self.history.extend(results)                         # ⑤ 追加到历史
```

#### ① 找工具

```python
tool = self.registry.get("get_weather")
```

进入 `registry.py`：

```python
def get(self, name: str):
    return self._tools.get("get_weather")  # 返回 WeatherTool 实例
```

`self._tools` 是在 `main.py` 中注册时构建的字典：

```python
registry.register(WeatherTool())
# → self._tools = {"get_weather": <WeatherTool 对象>}
```

#### ② 解析参数

```python
args = json.loads('{"city": "上海"}')
# → args = {"city": "上海"}
```

> **📌 知识点：json.loads()**
>
> LLM 返回的参数是 JSON **字符串**，需要用 `json.loads()` 解析成 Python 字典，才能作为关键字参数传给工具函数。

#### ③ ★ 真正执行工具

```python
content = tool.execute(**args)
# 等价于：tool.execute(city="上海")
```

`**args` 是 Python 的**字典解包**操作，把 `{"city": "上海"}` 解成 `city="上海"`。

进入 `WeatherTool.execute()`：

```python
# weather.py
def execute(self, city: str) -> str:
    resp = requests.get(
        f"https://wttr.in/{city}",     # → "https://wttr.in/上海"
        params={"format": "j1"},
        timeout=10,
    )
    resp.raise_for_status()
    data = resp.json()
    return self._format(data, city)
```

**实际发生的事情**：
1. 拼出 URL：`https://wttr.in/上海?format=j1`
2. 发 HTTP GET 请求到 wttr.in 服务器
3. 服务器返回 JSON 格式的天气数据
4. `_format()` 方法把 JSON 格式化成可读文本

返回结果示例：

```

💨 风速: 12 km/h  💧 湿度: 65%  👁 能见度: 10 km

  2026-05-12  晴  ↑30°C / ↓22°C
  2026-05-13  多云  ↑28°C / ↓21°C
  2026-05-14  小雨  ↑25°C / ↓19°C
```

#### ④ 收集结果

```python
results.append({
    "role": "tool",
    "tool_call_id": "call_abc123",    # 关联到 LLM 返回的那个 tool_call
    "content": "📍 上海天气\n🌡 当前: 28°C...",  # 工具返回的文本

```

#### ⑤ 追加到历史

```python

```

```python
[
    {"role": "user",      "content": "今天上海天气怎么样"},
    {"role": "assistant", "content": None, "tool_calls": [...]},
```

> **📌 知识点：tool_call_id 的作用**
>
> `tool_call_id` 把工具结果和 LLM 之前发起的工具调用关联起来。OpenAI 的 API 要求每个工具结果都必须对应一个 `tool_call_id`，这样 LLM 才能知道"这个结果是那个工具调用返回的"。

## 3. 重要知识点详解

### 3.1 三种消息角色（history 的构成）

| 角色 | 谁产生的 | 示例 |
|---|---|---|
| `"user"` | 用户 | `{"role": "user", "content": "今天上海天气怎么样"}` |
| `"assistant"` | LLM | `{"role": "assistant", "content": "上海今天28°C"}` |
| `"assistant"`（含 tool_calls） | LLM 要求调工具时 | `{"role": "assistant", "content": None, "tool_calls": [...]}` |
| `"tool"` | 工具执行结果 | `{"role": "tool", "tool_call_id": "xxx", "content": "28°C"}` |

**关键**：LLM 每次调用都看到完整的 history，所以能"记住"上下文。

### 3.2 OpenAI Function Calling 格式

```json
{
    "type": "function",
    "function": {
        "name": "get_weather",              // 工具名
        "description": "查询指定城市的天气",  // 工具说明
        "parameters": {                     // 参数定义
            "type": "object",
            "properties": {
                "city": {
                    "type": "string",
                    "description": "城市名称"
                }
            },
            "required": ["city"]
        }
    }
}
```

**LLM 通过这个 JSON 知道**：
- 有一个工具叫 `get_weather`
- 它的作用是查天气
- 调用时需要提供 `city` 参数（字符串类型，必填）

### 3.3 `**kwargs` 字典解包

```python
args = {"city": "上海"}
tool.execute(**args)
# 等价于：
tool.execute(city="上海")
```

**为什么需要解包？** 因为 LLM 返回的参数是 JSON 字符串，解析后是字典，而工具方法的参数是命名参数（`def execute(self, city: str)`），需要用 `**` 把字典解成关键字参数。

### 3.4 `model_dump()` 序列化

```python
self.history.append(msg.model_dump())
```

`msg` 是 OpenAI SDK 的 `ChatCompletionMessage` 对象，不能直接存进列表。`model_dump()` 把它转成普通的 Python 字典，才能追加到 `history`。

### 3.5 `tool_call_id` 关联机制

```
LLM 返回: tool_calls[0].id = "call_abc123"
                        ↓
工具结果: {"role": "tool", "tool_call_id": "call_abc123", ...}
                        ↓
LLM 下一轮看到: "哦，call_abc123 这个调用返回了 28°C"
```

### 3.6 钩子（Hook）机制

```python
def _emit(self, event: str, *args) -> None:
    if fn := self.hooks.get(event):  # 查找有没有注册回调
        fn(*args)                    # 有就调用
```

四个钩子：

| 钩子 | 触发时机 | 用途示例 |
|---|---|---|
| `on_start` | 用户消息刚进来 | 显示"正在思考..." |
| `on_think` | LLM 输出思考内容 | 显示推理过程 |
| `on_tool_call` | 即将执行工具 | 显示"正在查询天气..." |
| `on_reply` | LLM 给出最终回复 | 显示最终答案 |

**设计思想**：不修改核心代码，通过外部注入行为（开闭原则）。

### 3.7 `max_iterations` 安全防护

```python
for _ in range(self.max_iterations):  # 默认 5 次
```

防止 LLM 陷入无限循环（比如反复调用工具不给出答案）。超过次数返回超时提示。

## 5. 常见疑问与解答

### Q1: 为什么 LLM 能看懂工具定义？

因为 OpenAI 的 API 原生支持 `tools` 参数。LLM 在训练时已经见过大量类似的 JSON 格式，它能通过 `description` 理解工具用途，通过 `parameters` 知道需要什么参数。这就像给人类一张说明书，人类看了就知道怎么用。

### Q2: 为什么要把工具结果存进 history？

因为 LLM 是无状态的。如果不存，下一轮 LLM 就不知道工具返回了什么结果，也就无法基于结果回答用户。

### Q3: 为什么要有 max_iterations？

防止 LLM 陷入无限循环。比如 LLM 反复调用工具但不给出最终答案，或者工具调用链太长。默认 5 次是一个合理的上限。

### Q4: 钩子和直接改代码有什么区别？

- **直接改代码**：在 `_handle_tool_calls` 里硬编码 `print()` —— 不灵活，改核心代码风险大
- **钩子**：通过参数注入行为，核心代码不动 —— 灵活、可插拔、符合开闭原则

### Q5: 如果 LLM 返回多个 tool_calls 会怎样？

`_handle_tool_calls` 中的 `for tc in tool_calls` 会遍历所有工具调用，依次执行，结果一起追加到 history。LLM 下一轮能看到所有工具的结果。
